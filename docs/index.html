Introduction


aspLite is a lightweight framework for ASP/VBScript developers. It can help to develop better ASP/VBScript applications.

Installation:

Install as a website or virtual directory in IIS. The actual framework is located in /asplite/asplite.asp. The path /asplite can be changed to anything else in /asplite/config.asp. If you prefer to use aspLite in a subfolder /mypath/asplite, make sure to update config.asp correspondingly. To run the demo site that ships with the framework, you need to enable 32bits applications for your website's application pool, as it includes an Access database sample.

Highlights:

* split html/css/javascript and asp
* conditionally load asp code (rather than include it 'all')
* facilitate MVC application design with AJAX support
* charset utf-8
* .net-alike codebehind approach
* write your own plugins

Less is more

aspLite consists out of 2 files only (asplite.asp & config.asp file). The only thing this framework does is wrap useful ASP components into reusable functions:

* file streaming (text, code + binary)
* make use of VBScript executeGlobal
* ServerXMLHTTP & DOMDocument
* application caching 
* some generic ASP/VBScript functions

Get started

You may want to check demo.asp that comes with the framework. It sure will help to understand how aspLite works.

To start using aspLite, include it at very the top of your asp-page:
<!-- #include file="asplite/asplite.asp"-->

From then on, you can use an object of the aspLite class: aspL (it's already created). The word "aspL" should therefore be considered a reserved word. Do not re-declare, re-set or even destroy it. This would break aspLite.

Stick to 1 asp file for your project

There are some very good reasons to stick to 1 asp page only for your application, even if your application has 20 screens with 20 buttons or links each. And your best option is to use "default.asp" in the root of your application. "Default.asp" is always included in the list "default documents" in IIS. Finally, using only 1 asp page as suggested, keeps server memory usage by your web-app very low, specially when this technique is combined with the conditional load of asp scripts (as facilitated by aspLite).

Resx files?

As an ASP developer, you will quickly notice that resx files are used to include ASP scripts. Why? For security reasons. Resx files are never served nor executed by IIS. Therefore, it's impossible to inject or open resx files in the browser directly. Even though it's strongly recommended to use resx files, you *can* also use the regular .asp extensions for your handler-files. Make sure that visitors cannot inject (querystring, form submit) or load these pages in the browser directly.

Check the demo!

Again, check out demo.asp and its corresponding /code/demo.resx file on how aspLite can be used to organize your ASP application development.

Plugins

How to add plugins?

The following plugins are included already - but they are not necessary to use the framework i.e. they can be ignored. When no plugins are used, they will not affect the performance of your application in any way.

* connector for Access databases (OLEDB)
* mailer for CDO.Message
* jpg resize/recolor/crop (.net required)
* json read/write
* text/number randomizing
* rss feed reader

Bootstrap

aspLite does not include or favorize frontend CSS/JavaScript libraries. But Bootstrap and some AJAX examples both using jQuery are used in demo.asp. 

Roadmap

* make file (and complete folder?) uploads in ASP classic easier (core)
* keep security high (protect against sqli, csrf, brute force attacks, cookie theft, etc) (core)
* facilitate multilingual websites and applications (core)
* implement one or more WYSIWYG editors (as plugins)

Develop plugins for aspLite

aspLite will always have a very small footprint. You can however develop your own plugins for aspLite. How?

  1. Plugins are stored in the /asplite/plugins folder.
  2. Add a seperate folder for your plugin: eg: /asplite/plugins/myplugin/
  3. Add a resx file with the SAME name: /asplite/plugins/myplugin/myplugin.resx
  4. In myplugin.resx, create a class with this SAME name as follows:

    class cls_asplite_myplugin
    ...
    end class
  5. To create an instance of your plugin, use:

    dim myObj
    set myObj=aspL.plugin("myplugin")
    myObj.XX()

    You could also just:
    aspL.plugin("myplugin").XX()

    (where XX() is a public sub/function of your myplugin-class)
